High Level API Calls - Popup

High level calls are those that start with popup. The are the most basic form of communications with the user. The are named after the type of window they create, a pop-up window. These windows are meant to be short lived while, either delevering information or collecting it, and then quickly disappearing.

Think of Popups as your first windows, sort alike your first bicycle. It worked well, but was limited. It probably wasn’t long before you want more features and it seemed to limiting for your newly found sense of adventure.

For now, let’s stick with these 1-line window calls, the Popups. This is the list of popup calls available to you:

Popup Output

Think of the popup call as the GUI equivalent of a print statement. It’s you way of displaying results to a user in the windowed world. Each call to Popup will create a new Popup window.

popup` calls are normally blocking. Your program will stop executing until the user has closed the Popup window. A non-blocking window of Popup discussed in the async section.

Just like a print statement, you can pass any number of arguments you wish. They will be turned into strings and siplayed in the popup window.

There are a number of Popup output calls, each with a slightly different look or functionality (e.g. different button labels, window options).

The list of Popup output function are:

popup
popup_ok
popup_yes_no
popup_cancel
popup_ok_cancel
popup_error
popup_timed
popup_auto_close
popup_quick
popup_quick_message
popup_no_waitWait
popup_non_blocking
popup_notify

The trailing portion of the function name after Popup indicates what buttons are shown.

PopupYesNo shows a pair of button with Yes and No on them. PopupCancel has a Cancel button, etc.

While these are output windows, they do collect input in the form of buttons. The Popup functions return theh button that was clicked. If the Ok button was clicked, then Popup returns the string Ok. If theh user clicked the X button to close the window, then the button value returned is None or WIN_CLOSED is more explicit way of writing it.

The function popup_timed or popup_auto_close are popup windows that will automatically close after come period of time.

Here is a quick-reference showing how the Popup calls look.

sg.popup('popup')                       # Shows OK button
sg.popup_ok('popup_ok')                 # Shows OK button
sg.popup_yes_no('popup_yes_no')         # Shows Yes and No buttons
sg.popup_cancel('popup_cancel')         # Shows Cancelled button
sg.popup_ok_cancel('popup_ok_cancel')   # Shows OK and Cancel buttons
sg.popup_error('popup_error')           # Shows red error button
sg.popup_timed('popup_timed')           # Automatically closes
sg.popup_auto_close('popup_auto_close') # Same as PopupTimed

Popup - Display a popup Window with as many params as you wish to include. This is the GUI equivalent of the print statements. It’s also great for ‘pausing’ your program’s flow until the user can read some error messages.
