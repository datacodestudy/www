More about the Graph Element

Graph Element

All you math fans will enjoy this Element… an all you non-math fans willenjoy it even more.

I’ve found nothing to be less fun than dealing with a graphic’s coordinate system from a GUI Framework. It’s always upside down from what I want. (0,0) is in the upper left hand vorner… sometimes… or was it in the lower left?

How about the ability to get your own location of (0,0) and then using those coordinates instead of what thkinter provides? This results in a very powerful capability - working in you own units, and then displaying them in an area defined in pixels.

If you’ve ever been frustrated with where (0,0) is located on some surface you draw on, then fear not, your frustration ends right here. You get to draw using whatever coordinate system you want. Place (0,0) anywhere you want, including not anywhere on you Graph. You could definde a Graph that’s all negative numbers between -2.1 and -3.5 in the X axis and -3 to -8.2 in the Y axis.

There are3 values you’ll need to supply the Graph Element. The are:

  • Size of the canvas in pixels
  • The lower left (x,y) coordinate of you coordinate system
  • The upper right (x,y) coordinate of your coordinate system

After you supply those values you can scribble all of over your grah by creating Graph Figures. Graph Figures are created, and a Figure ID is obtained by calling:

  • DrawCircle
  • DrawLine
  • DrawPoint
  • DrawRectangle
  • DrawOval
  • DrawImage

You can move your figures around on the canvas by supplying the Figure ID the x,y delta to move. It does not move to an absolute position, but rahter an offset from where the figures is now. (Use Relocat to move to specific location)

graph.MoveFigure(my_circle, 10, 10)

You’ll also use this ID to delete individual figures you’ve drawn:

graph.DeleteFigure(my_circle)

Mouse Events Inside Graph Elements

If you have eneabled events for your Graph Element, then you can receive mouse click events. If you additionally enable drag_submits in you creation of the Graph Element, then you will also get events when you DRAG inside of a window. A Drag is defined as a left button down and then the mouse is moved. When a drag event happens, the event will be the Graph elemnts’s key. The value returned in the values dictionary is a tuple of the (x,y) location of the mouse currently. This means you’ll get a stream of events. If the mouse moves, you’ll get at LEAST 1 and likely a lot more than 1 event.

Mouse Up Event for Drags

When you’ve got drag_submits enabled, there’s a sticky situation that arises… what happens when you’re donw dragging and you’ve let got of the mouse button? How Mouse Up will generate an event to you with the value: Graph_key + +UP. Thus, if your Graph Element has a key of _GRAPH_, then the event you will receive when the mouse button is released is: _GRAPH_+UP.

Yea, it’s a little weird, but it works. It’s SIMPLE too. I recommend using the .startswith and .endswith built-ins when dealing with these kinds of string values.

Here is an example of the events and values dictionary that was generated by clicking and dragging inside of a Graph Element with the key == graph:

graph {'graph': (159, 256)}
graph {'graph': (157, 256)}
graph {'graph': (157, 256)}
graph {'graph': (157, 254)}
graph {'graph': (157, 254)}
graph {'graph': (154, 254)}
graph+UP {'graph': (154, 254)}
